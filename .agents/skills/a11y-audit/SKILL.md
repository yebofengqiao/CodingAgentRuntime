# a11y-audit

Use this skill when reviewing or implementing accessibility-sensitive UI changes.

- Start with semantic structure, focus order, and interactive affordances.
- Reuse the repository's existing component patterns before adding ARIA attributes.
- Read `references/checklist.md` when you need a fuller review checklist.
- Use `scripts/list_a11y_targets.sh` if you want a quick inventory of candidate files.
- `assets/review_template.md` can be copied into review notes or issue comments.
